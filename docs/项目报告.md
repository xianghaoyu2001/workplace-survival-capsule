## 一、研究目标、研究内容和拟解决的关键问题

### 1.1 研究背景与问题提出

职场能力评估是工业与组织心理学中的核心议题。传统评估方法存在三类长期未解决的矛盾。

**第一，标准化与生态效度的矛盾。** 自评量表（如 Big Five、EQ-i）和情境判断测验（SJT）虽具有标准化计分优势，但均以固定题本呈现——所有受测者面对相同的题目顺序和选项集合，无法根据受测者的实际作答动态调整后续提问。这导致评估脱离真实职场中"多角色、多轮次、动态追问"的交互特征，生态效度（ecological validity）受到根本性限制。

**第二，客观计分与评分者一致性的矛盾。** 评鉴中心（Assessment Center）通过多源评估者（Multi-Rater）观察受测者在模拟情境中的行为表现来评分，虽然生态效度较高，但严重依赖多位受过专业训练的评估师同时在场观察——这带来高昂的人力成本（每位受测者约 ¥500-3,000）和评分者间一致性（inter-rater reliability）的挑战。研究表明，即使经过标准化培训，不同评估师对同一行为的评分仍存在中等程度的变异（r 通常介于 0.4-0.6 之间）。

**第三，行为观察与评分溯源的矛盾。** 无论是传统评鉴中心还是计算机化 SJT，评分结果往往是概括性的等级评定（如 1-5 Likert 量表），缺乏对应受测者具体行为表现的原话记录。当评分结果受到质疑时（例如受测者申诉某维度分数偏低），评估方难以提供具体的、可追溯的行为证据来支撑评分决策。

本研究试图回答的核心问题是：**能否利用大语言模型的多智能体协同能力，构建一种同时具备评鉴中心的生态效度、标准化测验的计分一致性、以及行为证据可追溯性的新型职场能力评估工具？**

### 1.2 研究目标

**目标一：构建具有高生态效度的职场情境模拟系统。**

本研究旨在创建一种动态的、多角色交互的评估环境，使受测者沉浸于真实职场压力场景中，面对来自直属领导、平级同事和客户（或顾客）的交替追问。不同于固定题本的 SJT——所有受测者按相同顺序阅读相同的选项——本系统根据受测者的实时回答动态调整追问角色、追问焦点和压力强度。这种设计模拟了真实职场中"不同利益相关方基于同一事件从各自立场提出不同关切"的交互结构。评估不再是受测者"选择最佳答案"，而是在多轮压力追问中"构建并表达自己的解决方案"。

本系统预设两个职场场景：

- **场景A（项目演示危机）**：受测者扮演活动报名页项目负责人，明天需向客户演示。直属领导李总关心演示范围和风险边界、平级同事张伟担心短信故障风险被扩大为自己的全部责任、客户王经理关心明天能看到什么以及哪些风险需提前知道。能力维度：情绪控制、问题解决、风险沟通、协作边界、客户预期管理。轮次：6（最低）至 10（上限）。
- **场景B（咖啡店投诉）**：受测者扮演咖啡店值班店长，早高峰时段一位顾客因冰拿铁等候 18 分钟且标签贴错而要求退款并扬言在点评平台写差评。区域经理关心门店口碑与规则合规、新员工小林刚上岗情绪紧张需要保护、投诉顾客要求即时处理方案。能力维度：情绪克制、服务意识、规则边界、员工保护、现场秩序管理。轮次：8 至 12。

**目标二：建立可追溯、可校验的多维能力评分体系。**

本研究从心理测量学角度设计了三维评分保障机制：

1. **评分者信度保障**（对应传统评估中多位评分者独立打分后取一致性）：评分 Agent 对同一份对话记录进行三次独立评分（每次 temperature=0.1，以确保输出有微小但非零的变异），取中位数报告作为最终结果。三次分数方差存储为后续元分析材料。这模拟了传统评鉴中心中"三位独立评估师背对背评分、取共识分数"的流程，但消除了评估师个体差异对评分的影响。

2. **计分规则一致性校验**（对应传统测验中的计分标准核查）：自动校准器对中位数报告执行六项校验——总分是否等于五个维度分数之和（数学约束）、雷达图维度名称是否匹配场景预设的能力维度标签（语义约束）、雷达图各项分数是否等于对应维度分数（一致性约束）、证据引用条数是否不少于 1 条（充分性约束）、各维度分数是否在 0-20 的理论范围内（范围约束）。存在偏差时自动修复。

3. **行为锚定与证据可追溯性**（对应评鉴中心的行为锚定评定量表 BARS）：评估过程中，行为检测 Agent 对每轮受测者回答中的 21 项行为指标进行标注，每条正向标记附受测者原话引用。这些行为证据通过评估笔记累积传递到评分阶段——评分 Agent 被明确要求"每条 evidence 至少有对应的用户原话引用"。最终报告中的每项分数都可以追溯到具体轮次的具体回答。

**目标三：探索 AI 驱动的"多源反馈"评估模式的心理测量学特性。**

传统 360 度反馈评估（360-degree Feedback）依赖多位真实评分者（上级、同事、下属、客户）提供反馈，评分质量受评分者动机、人际关系、评分者培训程度等无关变量影响。本研究用三个独立的 AI 角色模拟者（Leader/Coworker/Client Agent）替代真实评分者进行行为追问——每个角色携带独立的信任度、压力感知、焦虑/防御状态等多轮记忆，从各自角色立场提出关切，但不直接参与评分。这种设计将"多源反馈"的评估结构从"多源评分"改为"多源追问+独立评分"，消除了评分者偏差（rater bias）的同时保留了多角色视角的评估丰富性。

### 1.3 研究内容

**（1）职场能力五维度模型的操作化定义。**

本研究基于工业与组织心理学中的胜任力建模（Competency Modeling）研究传统，综合评鉴中心常用的维度框架，定义了五个评估维度。每个维度有严格的操作化定义和互斥判别标准（避免评分者效应中的 halo effect）：

- **共情能力（Empathy）**：仅评估受测者是否识别并有效回应了不同角色的情感与关切（领导关心可控与合规、同事关心不被背锅与保护、客户/顾客关心范围与承诺依据）。不与组织视角混淆——共情看"看到了谁的担心"，不看"连接了哪些系统层面"。
- **情绪控制（Emotional Control）**：仅评估受测者在压力追问下是否保持结构化、不回避、不推责、不做无依据承诺。不与问题解决混淆——情绪控制看表达方式的稳定性，不看方案本身的质量。
- **问题解决（Problem Solving）**：仅评估受测者提出的方案是否可执行且完整——目标清晰、取舍合理、风险可控。优先评估方案质量本身，而非是否贴合预设路径。
- **谈判推进（Negotiation）**：仅评估受测者是否明确处理了同事/员工的合理责任边界——边界是否清晰、协作条件是否可接受、是否将整体结果压在同事/员工身上。
- **组织视角（Organizational Perspective）**：仅评估受测者是否连接了场景目标、团队/员工状态、客户/顾客影响和组织规则。若不能同时覆盖至少两个视角，此项最高 13 分。

每个维度的评分采用行为锚定评定量表（BARS）式描述：
- 0-4 分：严重失效（持续回避、甩锅、有害承诺）
- 5-8 分：明显不足（有态度但无具体动作、边界、风险说明或对象回应）
- 9-12 分：部分可用（有处理方向，但依据/步骤/边界/同步不完整）
- 13-15 分：稳定可用（明确行动、合理取舍、风险说明、至少一个关键对象的回应）
- 16-17 分：良好闭环（覆盖主要对象，说明责任边界、风险兜底和后续同步）
- 18-20 分：优秀（压力下稳定，主动澄清/取舍，完整覆盖目标/风险/边界/改进）

此外设有维度级评分上限作为区分效度的制度保障：受测者未明确处理同事/员工协作边界时，谈判维度最高 10 分；受测者未明确风险或兜底时，问题解决和情绪控制最高 12 分；受测者仅表态而无具体行动步骤时，所有维度最高 11 分。

**（2）动态行为检测系统的设计。**

本系统在每轮追问后对受测者的回答进行 21 项行为指标检测。这 21 项指标分为两类：

- **正向能力指标（13 项）**：是否识别核心流程、是否澄清了范围、是否区分为必须项和可选项、是否主动澄清未知信息、是否使用给定事实、是否标注假设、是否提出可行替代方案并说明前提和风险、是否承担推进责任、是否说明推进步骤、是否说明风险项、是否处理了同事/员工的责任边界与保护方式、是否管理了客户/顾客的范围和预期、是否形成闭环方案（范围+风险+协作边界+客户说明）。
- **负向行为指标（8 项）**：是否有过度承诺（无依据保证全部完成）、是否推责（将问题归咎于他人）、是否回避（面对追问时未正面回应）、是否过早给方案（未充分理解约束就泛化方案）、是否出现无依据假设（编造事实）、是否诚实说明风险并给出合理处理方式、是否主动承担推进责任、是否在压力下保持结构化表达。

每条正向标记附带受测者原话引用和判断理由，形成"行为观察记录"。这些记录通过评估笔记（evaluation_notes）累积传递，形成从受测者原话到最终评分的完整证据链。

**（3）AI 评分者的信效度评估。**

本研究首次系统性地关注 AI 作为评估者代理（AI as Rater Proxy）的心理测量学特性。评分 Agent 对同一份完整对话记录进行三次独立评分（每次 temperature=0.1 以确保评分过程存在微小但非零的差异），比较三次评分的一致性（对应评分者信度）和方差大小（对应评分精确度）。校准器在评分后自动执行六项规则校验（对应计分规则的一致性检查）。这些设计为后续大规模收集评分数据、进行评分者间信度（inter-rater reliability）和重测信度（test-retest reliability）的实证研究奠定了基础。

**（4）两个评估场景的心理测量学特性设计。**

两个场景覆盖不同的职场能力域和应激类型，旨在评估能力模型的跨情境稳定性（cross-situational consistency）：

- **项目演示危机**：属于"内部协作+外部客户交付"型职场应激。压力源包括时间压力（明天演示）、角色冲突（领导/同事/客户三方关切不同）、不确定性管理（短信功能不确定）。能力聚焦于范围界定、风险沟通、协作边界和客户预期管理。
- **咖啡店投诉**：属于"一线服务+现场应急处置"型职场应激。压力源包括时间紧迫性（早高峰排队）、情绪压力（愤怒顾客、紧张新员工）、利益冲突（退款要求 vs 门店规则、公开差评威胁）。能力聚焦于情绪克制、服务意识、规则边界、员工保护和现场秩序。

两个场景使用不同的能力维度标签，但均映射到相同的底层五维度理论框架（empathy/emotional_control/problem_solving/negotiation/organizational_perspective），使跨场景分数具备可比性——场景只影响各维度的权重而非底层能力定义。

### 1.4 拟解决的关键心理测量学问题

**问题一：如何提高 AI 情境测评的评分者信度（Inter-Rater Reliability）？**

在传统评鉴中心中，评分者间信度通常通过以下方式保障：多位评估师独立评分后计算 ICC（组内相关系数）或 Cohen's Kappa、评估师需通过标准化培训和认证。本研究的 AI 对应方案是：评分 Agent 对同一对话进行三次独立评分（temperature=0.1 保证评分过程有微小变化），取中位数报告为最终结果，记录三次分数的方差。方差超过预设阈值时，标记"建议人工复审"。六项校准规则自动校验评分的数学和格式一致性。未来的实证研究可通过计算三次模拟评分之间的 ICC 来量化 AI 评分者的评分者内信度（intra-rater reliability）。

**问题二：如何提高 AI 情境测评的维度区分效度（Discriminant Validity）？**

在传统 Likert 量表和评鉴中心中，halo effect（光环效应）是最常见的评分偏差——评估师对受测者的整体印象影响了各维度评分的独立性，导致各维度评分高度相关。本研究的应对策略是三管齐下：第一，每个维度有严格的操作化定义和互斥判别标准（共情只看关切识别 vs 组织视角看多视角连接；情绪控制看表达方式 vs 问题解决看方案质量）；第二，维度级评分上限制度强制区分度（未处理同事边界则谈判不得超过 10 分，未说明风险则问题解决和情绪控制不得超过 12 分）；第三，评分 Agent prompt 明确要求"维度是互斥判断，避免 halo effect"。未来的实证研究可通过计算五个维度间的相关系数矩阵、进行验证性因子分析（CFA）来量化区分效度的改善程度。

**问题三：如何平衡评估的标准化与生态效度？**

这是心理测量学中的经典 trade-off。标准化测验（如自评量表、固定 SJT）生态效度低但计分精确；评鉴中心生态效度高但标准化程度低且成本高。本研究的方案是通过"多源追问（生态效度源）+独立标准化评分（标准化源）"的架构分离来解决：三个 AI 角色模拟者在各自独立的心理状态（trust/pressure/anxiety/cooperation 等，1-5 分值）驱动下进行涉及角色压力的多轮追问——这是生态效度的来源；评分 Agent 在独立的评估环境中仅消费完整对话记录和行为证据——这是标准化的来源；而评分 Agent 的评分标准和行为检测规则对所有受测者和所有轮次完全一致——这是标准化的保障。

**问题四：如何让测评工具适应不同评估情境而不损失可比性？**

两个内置场景使用不同的能力维度标签（"情绪控制/问题解决/风险沟通/协作边界/客户预期管理" vs "情绪克制/服务意识/规则边界/员工保护/现场秩序管理"），但均映射到相同的五维度底层理论框架。评分 Agent prompt 中的评分锚点（0-4/5-8/9-12/13-15/16-17/18-20）和维度级上限规则跨场景完全一致。这种设计类似于心理测量中的"共同题设计"（common-item design）——通过共享的底层评分框架实现跨量表的等值（equating）。未来的实证研究可通过收集同一批受测者在两个场景下的得分，计算跨场景相关系数和 Bland-Altman 一致性分析来评估工具的跨情境稳定性。

## 二、研究方法与技术路线

### 2.1 评估框架设计：评鉴中心范式与 AI 的融合

本研究的方法论基础根植于工业与组织心理学中的**评鉴中心（Assessment Center）**范式。评鉴中心的核心要素包括：多位评估者从不同角度观察受测者、多种评估练习（exercises）模拟不同职场情境、评估维度预先定义、评分基于行为证据（而非特质推断）、最终分数由多位评估者讨论后达成共识。

本研究保留了评鉴中心的全部核心要素，但将人员替换为 AI Agent：

| 评鉴中心要素 | 传统实现 | 本研究的 AI 实现 |
|---|---|---|
| 多源评估者 | 2-4 位受过培训的人类评估师 | Director Agent（行为检测+评估编排）、Judge Agent（最终评分）、GroupDisc Agent（过程一致性校验） |
| 角色扮演者 | 受过培训的演员或评估师扮演同事/客户 | Leader/Coworker/Client Agent 各自维护独立的心理状态和记忆 |
| 评估练习 | 模拟会议、角色扮演、文件筐等 | 基于职场场景的 6-10 轮动态追问对话 |
| 行为观察与记录 | 评估师观察并记录行为实例 | BehaviorDetector 逐轮检测 21 项行为指标并附原话引用 |
| 评分整合 | 评估师讨论达成共识 | Judge 三次独立评分取中位数 + Calibrator 校验 |
| 反馈报告 | 书面评估报告 | 雷达图 + 五维度分数 + 证据引用 + 改进建议 |

### 2.2 共享状态黑板：评估过程的统一数据模型

所有评估数据存储在一个统一的"黑板"（Blackboard）数据结构中。黑板是受测者在测评过程中的完整心理测量档案，包含七个区块：

- **场景事实（scenario_facts）**：包括受测者角色与任务背景、9 条已知事实、4 条未知信息、3 条已完成功能、3 条风险功能、5 条行为约束、5 个能力维度标签、3 个利益相关方档案
- **评估目标（assessment_goals）**：9 项评估目标，对应五维度能力模型的操作化定义（理解事实、澄清未知、区分范围、提出方案、评估替代方案、分派责任与推进、处理同事边界、管理客户预期、收束为可执行判断）
- **能力进度（user_progress）**：13 项能力指标（每项为是/否），逐轮更新
- **最新行为检测（latest_user_behavior）**：8 项行为标记（过度承诺、推责、回避、过早给方案、无依据假设、诚实风险说明、主动承担、压力下结构化），每轮更新
- **角色记忆（npc_memory）**：三个角色各自的独立心理状态——Leader（信任度 1-5、压力 1-5、最近关切）、Coworker（合作度 1-5、防御度 1-5、最近关切）、Client（信任度 1-5、焦虑度 1-5、最近关切），多轮累积演化
- **评估笔记（evaluation_notes）**：优势记录、风险记录、行为证据引用，逐轮累积
- **评估控制（conversation_control）**：最近追问角色、建议下一角色、是否触发一致性校验、是否结束评估、结束原因

### 2.3 行为检测与评分流水线

每轮评估对话按以下流程执行：

1. **行为检测阶段**：Director Agent 读取黑板的场景事实、评估目标、当前能力进度和角色记忆状态，分析受测者的最新回答，输出 21 项行为标记（每条正向标记附受测者原话引用和理由说明），同时决定下一轮的追问角色（Leader/Coworker/Client 三者之一）、追问焦点（对应五维度中的某个待验证能力）、压力等级（1-5）以及具体的追问指令。

2. **角色交互阶段**：被选中的角色 Agent 先读取自己当前的记忆状态（信任度/合作度/防御度/焦虑度 + 最近关切），将心理状态转化为语气指令（例如信任度 ≤ 2→追问更尖锐、防御度 ≥ 4→先要求划清责任边界），然后按照追问指令向受测者发出 1-3 句回复。回复后基于受测者的回应质量更新自己的记忆状态（信任度/压力值 ± 1-2）。

3. **过程一致性校验**（条件执行）：当被触发时，GroupDisc Agent 以独立校验者身份检查三点——追问角色与追问焦点是否一致（如选择了 Coworker 追问但焦点是客户预期）、角色记忆状态是否与当前决策矛盾（如客户信任度很低但连续多轮未安排客户追问）、未解决事项是否与下一角色明显不匹配。无矛盾时返回"无需调整"（约 70% 轮次），有矛盾时输出修正指令。

4. **评分阶段**（评估结束时执行一次）：Judge Agent 读取完整对话历史、最终黑板状态和行为证据累积，三次并行独立评分（temperature=0.1），取 total_score 中位数报告。校准器自动校验六项规则并修复不一致项。最终输出包含总分、等级（高/中高/中/待提升）、五维度分数、维度分析、冲突处理风格、优势/风险/建议、行为证据引用和发展建议的完整评估报告。

### 2.4 评估质量保障的多层机制

**第一层：AI 评分者信度保障。** Judge Agent 对同一份对话进行三次独立评分，取中位数报告。这种设计模拟了传统心理测量中"三位独立评分者"的场景，但消除了评分者个体差异。三次分数方差在 0-5 分范围内（满分 100），超过阈值时标记"建议人工复审"。

**第二层：计分规则一致性保障。** 校准器在评分后自动执行六项规则校验：总分等于维度分之和、雷达图维度标签与场景预设一致、雷达图分数与维度分数一一对应、行为证据引用不少于最低条数、维度分数在理论范围内、存在偏差则自动修复。

**第三层：行为证据可追溯性。** 从行为检测阶段开始，每项正向行为标记附带受测者原话引用。这些引用通过评估笔记逐轮累积，最终传递到评分阶段。评分 Agent 被要求"每条 evidence 至少有对应的用户原话引用"、"优先引用用户原话，还必须引用评估笔记中记录的行为证据"。最终报告中的每项维度分数均可追溯到具体轮次的具体回答。

**第四层：评估过程完整性保障。** 角色回复越界检测（角色 Agent 不得替受测者写方案）→ 违规重试+安全兜底；评估环境一致性（同一场景所有受测者面对相同的事实集合、角色身份和评估目标）；动态适应性（追问路径因受测者而异但评估标准因受测者统一）。

### 2.5 技术实现概要

评估系统以 Express + TypeScript 为后端框架，SQLite 为数据存储，大语言模型 API（DeepSeek）为 Agent 的推理引擎。前端以 React + TypeScript + Vite 为框架，面向受测者提供完整的测评流程界面，面向评估管理员提供评估数据管理和可视化后台。系统含 25 个自动化测试用例覆盖核心评估逻辑。

## 三、预期结果与成果提供形式

### 3.1 预期评估产出

每位受测者完成一次评估后，系统产出以下心理测量学产品：

**（1）能力剖面图。** 雷达图展示五个维度的分数（0-20 分/维度），附数据表格和维度分析文字。总分范围 0-100 分，划分为四个等级（高 ≥85、中高 72-84、中 55-71、待提升 ≤54）。等级标签带有标准参照（criterion-referenced）含义——每个等级对应行为锚定描述（稳定可用/良好闭环/优秀等）。

**（2）维度分析报告。** 每个维度的评分理由，包括：该维度的操作化定义（确保受测者理解该分数"测量了什么"）、高分维度中支撑评分的具体行为实例（来自行为证据引用）、低分维度中缺失的能力表现（来自评估笔记的风险记录）。

**（3）行为证据链。** 对每条评分 16+ 的维度，提供至少一条行为证据引用（受测者原话 + 为什么该原话支撑该分数）。证据链从受测者在具体轮次中的具体回复→行为检测标记→评估笔记→最终评分，完整可追溯。

**（4）冲突处理风格评估。** 基于受测者在多轮交互中的整体行为模式，给出冲突处理风格标签（协作型/妥协型/回避型/竞争型/混合型），作为维度评分之外的质性补充。

**（5）发展建议。** 基于低分维度、行为风险记录和证据链，生成个性化的能力发展建议，指明"优先提升什么"和"怎么提升"。

### 3.2 预期心理测量学指标

基于系统的架构设计和评估逻辑的严谨性，提出以下实证研究假设供后续验证：

- **评估者内信度（Intra-Rater Reliability）**：三次 AI 评分间的 ICC（组内相关系数）预期 > 0.90，评分方差通常 < 5 分（满分 100）。方差 > 阈值时标记"建议人工复审"
- **维度区分效度（Discriminant Validity）**：五个维度间的 Pearson 相关系数预期 < 0.65（比较基准：传统评鉴中心中未经校正的维度间相关通常为 0.6-0.8）
- **证据充分性**：每条被标记为 16+ 的维度至少有 1 条行为证据引用，证据从受测者原话到最终评分的追溯链路完整
- **跨场景可比较性**：两个场景的不同维度标签均映射到相同的底层五维度理论框架，评分锚点和维度上限规则跨场景一致，使受测者在不同场景下的得分可进行有意义的比较
- **评估效率**：单次评估 6-10 轮交互，总时长约 8-15 分钟（取决于受测者作答速度），显著低于传统评鉴中心（通常 2-4 小时）。每位受测者的 AI API 调用成本约 ¥0.2-0.3（含并行三次采样评分），远低于人工评估

### 3.3 成果提供形式

1. **完整评估系统**：可运行的评估工具，含两个预设职场场景（项目演示危机、咖啡店投诉），SQLite 数据库零配置部署
2. **受测者评估报告**：雷达图 + 五维度分数 + 维度分析 + 优势/风险/发展建议 + 行为证据引用，面向受测者和管理者
3. **评估管理后台**：评估数据仪表盘（总人次/完成数/进行中/各维度均分）、受测者列表（按场景/状态筛选）、单人次完整评估回放（对话记录+可视化报告+行为证据链+评估过程记录）

## 四、特色与创新之处

### 4.1 "多源追问+独立评分"架构：评鉴中心与标准化测验的方法论融合

传统评鉴中心的多源评估（Multi-Rater Assessment）依赖 2-4 位人类评估师同时观察受测者行为并独立评分。这种设计存在方法论上的张力：多评估师提高了评估的全面性和公正性，但也引入了评估师间变异（inter-rater variance）和评分者偏差（rater bias——如严厉偏差、仁慈偏差、光环效应）。

本研究的核心方法论创新在于将评鉴中心的"多源"从评分层移到追问层。三个 AI 角色模拟者（Leader/Coworker/Client）各自携带独立的心理状态和角色立场，从不同角度对受测者施加压力追问——这保留了评鉴中心"多源视角"的评估丰富性。但最终的评分由独立的评分 Agent 在标准化条件下执行——评分标准、评分锚点和证据要求对所有受测者完全一致。这种"多源追问+独立评分"的架构分离消除了评分者偏差，同时保留了多角色交互的生态效度，是心理测量学方法论层面的有意义的探索。

### 4.2 行为锚定评分 + 证据可追溯链：从"给几分"到"为什么给这个分"

传统评估工具的评分结果（无论是 Likert 量表的数字还是评鉴中心的等级评定）往往只有结论没有过程——受测者、管理者或研究者无法追溯"这个分数基于什么得出"。这限制了评估结果的透明度、可解释性和申诉处理能力。

本研究从评估设计的根本上解决了这个问题。行为检测 Agent 在每轮对话中逐项标注 21 个行为指标，每个正向标记附带受测者原话引用和判断理由。这些行为证据通过评估笔记（evaluation_notes）逐轮累积传递到评分阶段——评分 Agent 被明确要求"必须消费评估笔记中的证据"、"每条 evidence 至少有一条对应的用户原话引用"。最终报告中，任何 ≥16 分的高分维度或 ≤6 分的低分维度都可以追溯到具体轮次中受测者的具体回答。

这本质上是将评鉴中心中"评估师在评分表上记录的行为实例（behavioral examples）"数字化、结构化、可检索化。它的心理测量学意义在于：当评分可追溯时，效度验证（validation）不再局限于统计指标（如维度间相关的模式），而是可以深入到"这个评分判断是否基于有效的、充分的、代表性的行为证据"的实质层面。

### 4.3 AI 评分者信度：多次采样取中位数的统计保障

在传统心理测量中，提高评分者信度的主要手段包括评分者培训（frame-of-reference training）、评分者数量增加（评分者越多，评分均值的信度越高）和评分标准细化（如 BARS 量表）。但这些手段均无法完全消除评分者个体差异（如人格特征、情绪状态、疲劳效应）。

本研究创造性地将统计方法（Bootstrap 式重采样）应用于 AI 评分场景。评分 Agent 对同一份对话记录进行三次独立评分——temperature=0.1 保证了每次评分的逻辑推理路径有微小差异但整体保持一致。取 total_score 中位数对应的完整报告作为最终结果。三次分数的方差量化了评分的不确定性，超过预设阈值（方差 > 某临界值）时标记"建议人工复审"。

这一设计的心理测量学意义在于：它不仅提高了评分的稳定性（消除了单次评分的随机波动），还量化了评分的可信度（方差本身就是评分质量的元指标）。当方差小时，管理者可对评分结果有高信心；当方差大时，系统主动提示需要人工介入。这种"评估结果 + 评分质量元数据"的双重输出，是心理测量报告形式的有益创新。

### 4.4 动态情境适应性：超越固定题本的交互式评估

传统 SJT（情境判断测验）的效度受限于其固定性——所有受测者阅读相同的场景描述并从相同的选项集中选择。这种"一次判断"的模式无法评估受测者"在被质疑、被追问、被施加新信息压力后是否能修正、调整或坚持自己的判断"，而后者恰恰是真实职场中最关键的沟通与应变能力。

本研究的评估过程是动态交互式的。行为检测 Agent 在每轮对话后更新受测者的能力进度和角色记忆状态。下一轮的追问角色、追问焦点、压力等级和追问指令都是基于"当前受测者的行为表现还存在哪些未验证的能力缺口"这一评估逻辑来动态选择的。这意味着两个受测者即使选择了同一场景、在首轮给出了相似的初始回答，后续的追问路径也可能完全不同（因为他们的行为特征和未验证缺口不同）。

从心理测量学角度看，这类似于计算机化自适应测验（CAT）的核心理念——根据受测者的反应动态调整后续测验项目的难度或内容，以达到高效精准的能力估计。但本研究将 CAT 从"选择下一道题"转化为"选择下一个追问策略"——这是一种新的自适应评估范式，可以称为"交互式自适应评估"（Interactive Adaptive Assessment）。

### 4.5 评估场景跨情境可比性：共享底层维度框架的等值设计

两个评估场景使用不同的能力维度标签：项目场景使用"情绪控制/问题解决/风险沟通/协作边界/客户预期管理"，咖啡店场景使用"情绪克制/服务意识/规则边界/员工保护/现场秩序管理"。这些中文标签分别适配了两个场景的职场语境，但均映射到相同的五维度底层理论框架（empathy/emotional_control/problem_solving/negotiation/organizational_perspective）。

这类似于心理测量中的等值设计（equating design）——不同测验版本通过共享的底层测量结构实现分数的跨版本可比较性。本研究的评分 Agent prompt 中的 0-20 分评分锚点、维度级评分上限规则、互斥维度定义和证据要求在所有场景中完全一致。这种设计使我们可以收集同一批受测者在不同场景下的得分数据，计算跨场景的相关系数（评估测量不变性 measurement invariance），以及分析不同能力在跨情境条件下的一致性与变异性——这本身就是一个有心理学价值的实证研究问题。

### 4.6 评估质量的元分析能力：内置的评分质量追踪系统

传统心理测量工具的效度验证通常需要事后设计专门的效度研究（如通过另一次大规模施测收集效标关联效度证据）。本研究在评估系统的架构中内置了心理测量学元数据的自动收集机制。每次评估完成后，后台自动记录：三次 AI 评分的具体分数和方差、各维度分数间的相关模式（如 halo effect 可操作化为维度间相关系数的高低）、行为检测 Agent 标记的行为模式与最终评分的一致程度。这些数据随着评估人次的积累而自然增长——当数据量超过 50 份时，可以进行探索性因子分析（EFA）检验五维度模型的因子结构是否稳定；当数据量超过 200 份时，可以计算效标关联效度（如与传统评鉴中心分数或工作绩效评分的相关性）。这种"评估工具内置效度验证数据生产线"的设计，使得效度验证从一次性研究变为持续性的质量监控过程。

## 五、成本预算

### 5.1 评估实施成本

每位受测者单次评估的 AI API 调用构成：行为检测与评估编排约 8 次、角色追问约 8 次、过程一致性校验约 2 次（条件触发）、最终评分 3 次（并行采样）——合计约 22 次 API 调用。以 DeepSeek API 的当前公开定价计算，**每人次评估成本约 ¥0.2-0.25 元**。

相比传统评鉴中心（每人次 ¥500-3,000，含多位评估师的人力成本和场地成本）和商用在线测评平台（每人次 ¥10-50 的 SaaS 授权费），MAAT 系统的边际成本具有数量级上的优势。

### 5.2 系统部署与运营成本

单台轻量云服务器（2C4G，SQLite 本地数据库）年费约 ¥660-1,320 元，可支撑日均数百次的测评并发。无需独立数据库服务器、无需专业运维。若每年施测 10,000 人次，人均总成本（含服务器分摊和 API 调用）约为 ¥0.27-0.38 元。

### 5.3 工具开发成本

五维度能力模型的定义与操作化、评估场景的设计与验证、行为检测标准的制定与迭代、评分规则的校准——这些心理测量学核心工作的开发总量约为 49 人天（约 2.3 人月）。其中，AI 辅助编码工具（Claude Code）承担了约 60% 的重复性工作（Prompt 迭代、代码生成、文档撰写、测试用例编写），有效降低了人工投入。

### 5.4 后续效度验证研究预算建议

- **评分者信度实证研究**（30-50 份对话样本，由 3 位人类评估师和 AI 评分者独立评分，计算 ICC）——约 ¥2,000-3,000（人类评估师劳务费）
- **效标关联效度研究**（收集受测者的工作绩效评分或其他已验效的心理测量工具得分，与本系统分数进行相关分析）——约 ¥3,000-8,000（取决于样本量和效标数据获取难度）
- **因子结构验证**（收集 200+ 份评估数据后进行 CFA）——约 ¥500-1,000（数据分析人力）
- **跨场景等值研究**（同一批受测者完成两个场景，计算跨场景的测量不变性）——约 ¥1,000-2,000
